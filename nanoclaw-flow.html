NanoClaw Orchestration Flow
Sid's Dev Workflow — Full Cycle v1.0
Phase 1 — Foundation
01
Write the PRD in Perplexity (Comet)
Define your project — features, goals, user flows, tech stack. Be specific. The richer the PRD, the better every downstream prompt will be.
Perplexity + Comet
02
Generate Sequential Coding Prompts
Feed the PRD into Perplexity (Comet). Ask it to return a JSON array of sequential coding prompts — one per feature or screen. NanoClaw stores and queues these.
Perplexity + Comet NanoClaw stores output
Phase 2 — UI Design
03
Generate Per-Screen Prompts for Stitch
Ask Perplexity to generate a separate UI prompt for each screen of your app — tailored for Stitch. One prompt per screen, describing layout, components, and purpose.
Perplexity + Comet
04
Design UI in Stitch → Download ZIP
Open Stitch, paste each screen prompt, generate the UI. Download the full ZIP. NanoClaw automatically extracts it into the project folder and reads all HTML files.
Stitch NanoClaw extracts ZIP
Phase 3 — Iterative Build Loop
05
OpenCode Reads HTML Files from Stitch
NanoClaw instructs OpenCode to read all extracted HTML files. OpenCode understands the full UI structure and is ready to integrate everything into the codebase.
OpenCode NanoClaw orchestrates
06
Perplexity ↔ OpenCode Build Loop
The core iterative cycle. NanoClaw runs this automatically — waiting 1 min between steps, extending to 3 min on timeout.
Repeating Build Cycle
→
Perplexity (Comet): Given PRD + last OpenCode output, return next coding prompt
→
NanoClaw: Passes prompt to OpenCode, waits for output (1–3 min adaptive timeout)
→
OpenCode: Executes prompt using MiniMax M2.5 (free) — returns code output
→
NanoClaw: Saves iteration to project memory JSON file
→
Telegram: Sends step completion update to @Siddharth0001
↻
Repeat until Perplexity returns PROJECT_COMPLETE
Perplexity → OpenCode → Memory → Telegram → repeat
Phase 4 — Analysis & Polish Loop
07
Antigravity Deep Analysis — Claude Opus 4.6
Open Antigravity, select Claude Opus 4.6. Feed in the completed project. Antigravity produces a structured report: what's done, what's missing, bugs, and prioritized improvements.
Antigravity Claude Opus 4.6
08
Refinement Loop — Antigravity → Perplexity → OpenCode
Feed Antigravity's analysis back into Perplexity. Get new prompts. Run OpenCode again. Repeat 2–3 cycles until the project is professional-grade.
Perplexity + Comet OpenCode Antigravity
Refinement Cycle
→
Antigravity (Opus 4.6): Analyze project → output requirements + improvement doc
→
Perplexity (Comet): Take analysis → generate refined coding prompts
→
OpenCode: Run refined prompts → improved codebase
↻
Repeat 2–3 cycles until project is professional-grade
Antigravity → Perplexity → OpenCode → repeat
09
Finalize, Save Memory & New Project
Project complete. NanoClaw saves the full memory JSON. Telegram sends final notification. For next project, run with -new flag to wipe memory and start fresh.

Step 3 — Per-Screen Prompts for Stitch
Stitch
Given this PRD: [PASTE YOUR PRD HERE] Generate one UI design prompt per screen for use in Stitch (Google UI generator). Each prompt should describe: screen purpose, layout, key components, visual style. Return as a numbered list. Be specific enough that Stitch generates a complete, usable screen from each prompt alone.
Step 6 — Next Build Prompt (Loop)
Perplexity + Comet

Antigravity
